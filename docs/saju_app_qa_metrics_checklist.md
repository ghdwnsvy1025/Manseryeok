## 3. 카테고리와 일기 UX

- [ ] 최초 온보딩에서 4개 미만 선택을 막는다.
- [ ] 9개 초과 선택을 막는다.
- [ ] 기본 추천 6개가 올바르게 표시된다.
- [ ] 카테고리 순서 변경이 저장된다.
- [ ] 카테고리 비활성화 후 과거 데이터가 남아 있다.
- [ ] 재활성화 시 기존 기준선과 기록을 재사용한다.
- [ ] 모든 점수는 1~5 범위만 허용한다.
- [ ] 기본 점수가 미리 선택되지 않는다.
- [ ] `해당 없음`이 3점이 아닌 null로 저장된다.
- [ ] 일부 카테고리만 입력해도 저장 정책에 맞게 처리된다.
- [ ] 날짜와 사용자 timezone이 함께 저장된다.
- [ ] 네트워크 재시도에서 중복 일기가 생기지 않는다.
- [ ] 사건 태그는 0개여도 저장 가능하다.
- [ ] AI 제안 태그와 사용자 확정 태그를 구분한다.
- [ ] 일기 수정 시 모델 데이터가 갱신 또는 재학습 대상으로 표시된다.

## 4. 점수 분포와 정규화 검증

### 단위 테스트

- [ ] 기록 0개에서 오류 없이 fallback한다.
- [ ] 기록 13개에서는 개인 기준선 확정 상태가 아니다.
- [ ] 기록 14개에서 기준선 계산이 시작된다.
- [ ] 모든 값이 3일 때 0으로 나누지 않는다.
- [ ] 표준편차가 0.5 미만이면 분모 floor 0.5를 사용한다.
- [ ] normalizedZ가 -2.5보다 작으면 -2.5로 제한된다.
- [ ] normalizedZ가 2.5보다 크면 2.5로 제한된다.
- [ ] 최근 기록이 과거 기록보다 높은 가중치를 갖는다.
- [ ] `해당 없음`은 평균과 표준편차 계산에서 제외된다.
- [ ] 카테고리별 기준선이 독립적이다.
- [ ] 사용자별 기준선이 독립적이다.

### 운영 품질 경고

다음 조건을 감지하고 내부 품질 상태에 기록한다.

| 경고 코드 | 조건 | 영향 | 사용자 표시 |
|---|---|---|---|
| insufficient_samples | 유효 기록 < 14 | 개인 모델 미생성 | 기록이 더 필요함 |
| low_variance | 최근 14개에서 서로 다른 점수 ≤ 2 | 회귀 신호 약함 | 점수 범위를 조금 더 구분해 기록하도록 부드럽게 안내 |
| high_missingness | 최근 30일 완료율 < 40% | 신뢰도 하락 | 데이터 부족 |
| stale_data | 마지막 기록 후 30일 초과 | 현재성 하락 | 최근 기록 필요 |
| outlier_heavy | 극단값이 결과를 과도하게 지배 | 모델 불안정 | 신뢰도 낮춤 |

사용자에게 점수를 억지로 고르게 분산시키라고 요구하지 않는다.

## 5. 사주 계산과 특징 스냅샷

- [ ] `viewMode=diary_detail`은 `native_with_luck`이다.
- [ ] `viewMode=simple`은 `luck_only`이다.
- [ ] `calculationMode` 직접 입력이 viewMode보다 우선한다.
- [ ] 입력이 없으면 `native_with_luck`가 기본이다.
- [ ] 원국 입력 순서가 기존 명세와 일치한다.
- [ ] 지지는 대표오행 하나가 아니라 기존 지장간 기반 계산을 사용한다.
- [ ] 오행 percentage 출력 순서가 목·화·토·금·수로 일정하다.
- [ ] percentage 합계가 허용 오차 안에서 100이다.
- [ ] raw, percentage, detail이 보존된다.
- [ ] 계산 버전이 저장된다.
- [ ] 동일 hash의 스냅샷은 재사용된다.
- [ ] 계산 버전 변경 시 새 스냅샷이 생성된다.
- [ ] 원국과 운의 해석이 화면에서 구분된다.
- [ ] 오행 수치만으로 최종 문장을 결정하지 않는다.

## 6. 개인화 모델 QA

### Cold start

- [ ] 0~13개 기록에서는 모델이 생성되지 않는다.
- [ ] 이 단계에서 예측 숫자가 노출되지 않는다.
- [ ] 이론 요약과 단순 기록 추세만 표시된다.

### Early model

- [ ] 14번째 유효 기록 이후 카테고리별 모델이 생성된다.
- [ ] 기본 특징은 오행 5개와 십신 기능축 5개다.
- [ ] 높은 Ridge 규제가 적용된다.
- [ ] 결과가 `초기 경향`으로 표시된다.

### Active model

- [ ] 30개 이상에서 시간 순서 holdout 검증을 수행한다.
- [ ] 데이터 순서를 무작위로 섞지 않는다.
- [ ] 결측 target은 학습에서 제외된다.
- [ ] 카테고리마다 모델이 독립적이다.
- [ ] feature order와 model version이 저장된다.
- [ ] 동일 모델과 입력의 예측이 재현된다.
- [ ] 마지막 20% 데이터가 평가용으로 분리된다.
- [ ] 45개 이상일 때만 lambda 후보 비교가 실행된다.
- [ ] 기준선 MAE보다 나쁜 모델은 degraded 처리된다.
- [ ] degraded 모델의 예측이 사용자에게 노출되지 않는다.
- [ ] feature/calculation 버전 변경 시 재학습된다.
- [ ] 수정·삭제된 일기가 학습 데이터에서 올바르게 반영된다.

### 모델 평가 표

| 사용자/코호트 | 카테고리 | N | 모델 상태 | Baseline MAE | Model MAE | 개선율 | 방향 정확도 | Spearman | 신뢰도 |
|---|---|---:|---|---:|---:|---:|---:|---:|---:|
|  |  |  |  |  |  |  |  |  |  |

권장 해석:

```text
MAE 개선율 = (Baseline MAE - Model MAE) / Baseline MAE
방향 정확도 = 실제 점수가 개인 기준선보다 위/아래인지 맞힌 비율
```

출시 초기 내부 목표 예시:

- 학습 가능 사용자 중 50% 이상에서 기준선 대비 MAE 비악화
- active 모델 중 60% 이상에서 방향 정확도 55% 이상
- degraded 모델 노출률 0%

이 값은 운영 데이터에 따라 조정하며, 달성하지 못했다고 사주 이론의 참·거짓으로 해석하지 않는다.

## 7. 신뢰도 검증

- [ ] 표본 수가 증가하면 volumeScore가 단조 증가한다.
- [ ] 입력 누락이 많으면 coverageScore가 하락한다.
- [ ] 점수 변화가 거의 없으면 variationScore가 하락한다.
- [ ] 최근 기록이 없으면 recencyScore가 하락한다.
- [ ] 기준선보다 나쁜 모델은 validationScore가 높게 나오지 않는다.
- [ ] 계수 방향이 자주 바뀌면 stabilityScore가 하락한다.
- [ ] 신뢰도 85 이상이어도 확정 예언 문구가 나오지 않는다.

## 8. 분석 문장 QA

각 샘플 응답에 대해 아래 항목을 검수한다.

- [ ] `명리 이론상` 문단이 있다.
- [ ] `내 기록상` 문단이 있다.
- [ ] `실천 제안` 문단이 있다.
- [ ] 사용한 표본 수가 표시된다.
- [ ] 신뢰도가 표시된다.
- [ ] 원국과 운이 구분된다.
- [ ] 계산 수치가 해석 전체를 지배하지 않는다.
- [ ] “반드시”, “확실히 발생”, “운명적으로” 같은 단정 문구가 없다.
- [ ] 상관 패턴을 원인이라고 표현하지 않는다.
- [ ] 건강을 의학적 진단처럼 표현하지 않는다.
- [ ] 죽음·사고·파산·결혼을 확정 예언하지 않는다.
- [ ] 데이터가 부족하면 부족하다고 명시한다.
- [ ] 사용자에게 현실적인 작은 행동을 제안한다.

### 금지 문장 자동 테스트 예시

```text
반드시 사고가 난다
큰 병이 생긴다
파산한다
결혼하게 된다
이혼한다
사주 때문에 우울하다
이 수치가 운명을 증명한다
```

문맥 검수도 필요하므로 단순 금칙어 필터만으로 완료 처리하지 않는다.

## 11. 제품 운영 지표

### 활성화

```text
온보딩 완료율 = 카테고리 선택 완료 사용자 / 온보딩 시작 사용자
첫 기록 완료율 = 가입 후 24시간 내 일기 저장 사용자 / 신규 가입 사용자
```

### 유지율

```text
D7 기록 유지율 = 첫 기록 후 7일째 ±1일에 기록한 사용자 / 첫 기록 사용자
D30 기록 유지율 = 첫 기록 후 30일째 ±3일에 기록한 사용자 / 첫 기록 사용자
```

### 기록 품질

```text
카테고리 완료율 = 입력된 활성 카테고리 수 / 해당 일자의 활성 카테고리 수
태그 사용률 = 태그가 1개 이상인 일기 / 전체 일기
텍스트 사용률 = 자유 일기가 있는 기록 / 전체 일기
```

### 개인화 도달률

```text
Early 도달률 = 14개 이상 유효 기록 사용자 / 기록 사용자
Active 도달률 = 30개 이상 유효 기록 사용자 / 기록 사용자
Stable 도달률 = 90개 이상 유효 기록 사용자 / 기록 사용자
```

### 모델 유효성

```text
모델 활성률 = active 상태 모델 / 학습 시도 모델
기준선 개선률 = 기준선보다 MAE가 좋은 모델 / 평가 가능한 모델
예측 노출률 = 예측을 실제 표시한 분석 조회 / 전체 분석 조회
```

Legacy 진입률

기존 만세력 계산 실패율
